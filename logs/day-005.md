## What I Did
Completed the OhSINT room on TryHackMe. Starting point was a single
image file ( The famous Windows XP wallpaper 'Bliss'. Used EXIF metadata extraction to pull the copyright
holder and GPS coordinates embedded in the image. The copyright holder was OWoodFlint.From there,
pivoted to social media OSINT on the identified individual — O.Wood
Flint — tracing their presence across X(Twitter), GitHub, and WordPress.
Extracted public information as tasked by TryHackMe like his profile picture (A cat), his email, current travelling location, and
SSID from the BSSID meentioned in his X post. Had to perform a search on WiGle for that. Final flag was a password hidden in white text on a white
background on their WordPress site. I simply pressed select all once and it showed after some ignorance and efforts.

## What Clicked
EXIF data is an underestimated attack surface. A single image posted
publicly can leak GPS coordinates, device model, timestamp, and
copyright identity — none of it visible to the naked eye. Most people
have no idea their photos carry this metadata.

The pivot chain is the real OSINT skill — one data point leads to
another. Username on Twitter → same username on GitHub → WordPress
blog → hidden text. Each platform leaks something different about
the same person.

## What Confused Me
BSSID lookup via WiGLE required account registration and felt
like a dead end initially. The correlation between a publicly
broadcast BSSID and physical location is powerful but depends
entirely on WiGLE's crowdsourced database coverage — sparse areas
would make this technique unreliable.
