# Day 006 — How Websites Work & Web Application Basics

**Source:** TryHackMe

## What I Did
Covered the foundational mechanics of how websites and web
applications function — HTML structure, how browsers request and
render pages, and the client-server model that underpins all
web-based targets.

## What Clicked
This isn't web development training — it's reconnaissance context.
Before you can attack a web application, you need to understand
what's actually being served to the browser, what lives client-side
vs server-side, and where the boundary between the two creates
exploitable gaps.

HTML is the surface. What matters in pentesting is what's hidden
beneath it — server responses, input fields, cookies, and how the
application handles data it receives.

## What Confused Me
Nothing technically complex at this stage — this was prerequisite
groundwork. The real questions will emerge once active scanning
begins.

## Concept of the Day
Client sends HTTP request → Server processes → Server sends response
What the server does with your input is where vulnerabilities live.
